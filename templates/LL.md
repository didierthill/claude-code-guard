# LL.md — Lessons Learned

> Document every significant agent mistake, infrastructure gotcha, or non-obvious behavior.
> Read this before starting work. Update after every incident.
> Format: structured entries with date, symptom, cause, impact, fix, and rule.

---

## LL-001: Template — Copy this for new entries

**Date**: YYYY-MM-DD
**Symptom**: What went wrong (observable behavior).
**Cause**: Why it happened (root cause).
**Impact**: What it cost (time, broken builds, user impact).
**Fix**: What was done to resolve it.
**Rule**: What rule prevents this from recurring.
